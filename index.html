System Overview

Autonomy logic with real-time constraints.

The project models a compact self-driving controller themed around Luminar Technologies. User input changes the maximum requested speed, which increases or decreases the simulated perception workload. A hard real-time brake monitor watches a sliding potentiometer and triggers an emergency LED response when the threshold is crossed.

The system can be toggled locally through a physical button interrupt or remotely through a Wi-Fi web server. The web server also reports speed request, emergency brake state, and self-driving state.

Scheduler Map

Task placement across both ESP32 cores.

Core 1 handles control logic, button interrupts, braking, heartbeat, and simulated perception. Core 0 hosts the Wi-Fi web server and dashboard endpoint work.

Core 1 · Control

Heartbeat Priority 1 · soft

Power indication through a periodic heartbeat task.

Speed Input Priority 1 · soft

Reads a potentiometer to adjust maximum requested speed.

Perception Load Priority 2 · soft

Simulates the number of outside objects being processed.

Brake Monitor Priority 3 · hard

Watches a sliding potentiometer and signals emergency braking.

Event Handler Priority 3 · hard

Handles toggle events and emergency brake semaphore responses.

Button ISR interrupt

Toggles the self-driving system from a physical button.

Core 0 · Network

Web Server Dashboard

Reports max speed request, emergency brake state, system state, and accepts remote toggle commands.
